Design Goals

AKA: biting the bullet

The Basics

Provide an intuitive query langauge: inspired by the MongoDB gold standard

Must be Node-y: callback-based queries and promise-based cursors - we got'em

Low learning curve: consistent API across Browser and Node.js

Backbone.js Model and Collection compatible with built-in relationships: we use Knockback.js

Trivial JSON API generation: raw or custom JSON DSLs

Above and Beyond

URL-based query syntax: debug JSON queries on the search bar

JSON and Model interfaces for JSON-only APIs when a full relation, model-based ORM is overkill

Batch-based, browser-side processing of server models

Node.js-based stream support: anyone want to stream JSON from Postgres?

Relationships in MongoDB: it came for free

In Memory store: not all data needs to persist
